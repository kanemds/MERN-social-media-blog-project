Installing with npm i -g:

Installs the package globally on your system.
Makes the package accessible from anywhere in your system, not specific to a project.
Typically used for command-line tools or utilities.
Allows running package commands from the command line regardless of the working directory.

npm update -g:

updates globally installed packages on your system.

================================================================================================================

Installing with npm i -D:

Installs the package as a development dependency.
Lists the package in the devDependencies section of your project's package.json file.
Used for packages needed during the development process.
Examples include testing frameworks, build tools, or linters.

npm update -D:

updates development dependencies within your project.
